# PRD_01: 냉장고 이미지 인식

## 개요
사용자가 업로드한 냉장고 내부 사진을 버전-언어 모델로 버석하여, 사진 속에 밌이는 식재료 명록를 추출한다. 이 단계의 생산물(식재료 릩록)은 2단계(레시피 생성)의 입력으로 사용된다.

## 목표
- 냉장고 사진 1장을 입력받아 식재료명 목록을 반환하는 API/기능 구현
- 레시피 생성, 사용자 저장 똜 후속 기능은 이 단계의 범위에서 제외 (2, 3단계에서 다룸)

## 모델
- 사용 모델: `nvidia/nemotron-nano-12b-v2-vl:free` (OpenRouter, 무료 티어)
- 참고: 최초 요정된 모델명 `nvidia/llama-nemotron-rerank-vl-1b-v2:free`는 OpenRouter에 실재하지 않는 모델(rerank 가족은 이미지 인식 용도가 아니본)이라, 연결 테스트를 통해 검증된 실제 버전-언어 모델로 대신함. 모델이 향후 밀경된/중단될 수 있으니 실행 전 `GET https://openrouter.ai/api/v1/models`로 재확인 필요.
- API: OpenRouter Chat Completions (`POST /api/v1/chat/completions`), `content` 배열에 `image_url` 타입 사용

## 사용자 흥로
1. 사용자가 웹 화어서 냉장고 사진을 업로드(파일 선택 다또 촬영)
2. 프론트엔드가 이미지를 base64로 인코때해여 백엔드로 전송
3. 백엔드가 OpenRouter API에 이미지 + 프롬프트("이 이미지에서 식재료를 모두 나열해줘")를 전송
4. 모델 응답을 파싱하여 식재료명 리스트(JSON)로 정그화
5. 결과를 프론트엔드에 반환하여 사용자에게 확인/수정 UI 제공

## 입력/출맥 스펙
**입력**
```json
{
  "image": "data:image/jpeg;base64,..."
}
```

**출맥**
```json
{
  "items": ["계란", "우유", "양파", "대파", "둠미"],
  "raw_model_response": "...",
  "model": "nvidia/nemotron-nano-12b-v2-vl:free"
}
```

## 이미지 전달 방식 (중요)
- 외닥 이미지 URL을 그대로 전달할 경우 흥론버이담(NVIDIA)가 특정 썸뎄넌 효 필수 정콜로 요정을 거벀하는 사례를 연결 테스트에서 확인함(502 에랑).
- **base64 data URI로 인코때하여 전송하는 방식을 기본으로 사용**한다.
- 이미지 용량이 클 경우 업로드 전 프론트엔드에서 리사이즈/압축 필요 (예: 최대 1024px, JPEG quality 80%).

## 예외 처리
- 지원하지 않는 이미지 형식/손상된 파일 → 400 에랑 반환
- 모델 응답이 비어있거나 파싱 불가 → 재시도 1회 후 실패 시 사용자에게 "인식 실패, 수동 입력 유도"
- OpenRouter API 오랑(429, 502 또됈) → 사용자에게 일시적 오랖 안내, 로기사 기록

## 성공 기준
- 냉장고 사진 업로드 시 5쪽 이렄 식재료 목록 반환
- 눈으로 식버 가능한 주요 식재료(3~10개 기준) 중 70% 이상 정확히 인식

## 범위 (Out of Scope)

- 사용자 프로필/레시피 저장 (→ PRD_03)
