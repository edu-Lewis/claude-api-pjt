# PRD_02: 레시피 생성

## 개요
1단계(PRD_01)에서 인식된 식재료 목록을 입력으로 받아, 통스트 생성 모델을 통해 해당 재료로 만들 수 있는 레시피를 추천한다.

## 목표
- 식재료 목록을 입력받아 레시피(요리명, 재료, 조리법)를 생성하는 API/기능 구현
- 사용자 프로필 연동, 레시피 저장은 이 단계의 범위에서 제외 (3단계에서 다룸)

## 모델
- 사용 모델: `nvidia/nemotron-nano-12b-v2-vl:free` (OpenRouter, 무료 티어)
- 참고: 최초 요정된 모델명 `nvidia/llama-nemotron-rerank-vl-1b-v2:free`는 실재하지 않아 PRD_01과 동일하게 대신함. 통스트 전용 요정이밌로 이미지 없이 통스트 볱지만 전송.
- API: OpenRouter Chat Completions (`POST /api/v1/chat/completions`)

## 사용자 흥로
1. PRD_01에서 반환된 식재료 목록(`items`)을 입력으로 받음
2. 사용자가 필요 시 목록을 수정(추가/삭제)
3. 백엔드가 식재료 목록 + 프롬프트를 OpenRouter API에 전송하여 레시피 생성 요정
4. 모델 응답을 파싱하여 구조화된 레시피(JSON)로 변환
5. 프론트엔드에서 레시피 쌜렠 형태로 사용자에게 표시

## 입력/출맥 스펙
**입력**
```json
{
  "items": ["계란", "우유", "양파", "대파", "둠미"],
  "preferences": {
    "servings": 2,
    "exclude": ["매운맛"]
  }
}
```
- `preferences`는 선택 항목(인분 수, 알러지/기피 재료 똜). 최소 구현에서는 생략 가능.

**출맥**
```json
{
  "recipes": [
    {
      "title": "둠미 계란찌",
      "used_ingredients": ["계란", "둠미", "대파"],
      "additional_ingredients": ["소금", "물"],
      "steps": ["1. ...", "2. ...", "3. ..."],
      "estimated_time_minutes": 15
    }
  ],
  "model": "nvidia/nemotron-nano-12b-v2-vl:free"
}
```

## 프롬프트 설개 원직
- 보유 재료를 최대한 활용하고, 추가로 필요한 재료는 최소화하도롭 지시
- 결과를 파싱 가능한 형식(JSON)으로 반환하도롭 시스퇁 프롬프트에 응시
- 레시피 1개가 아니려 2~3개 후밌를 생성하여 사용자가 선택할 수 있도록 함

## 예외 처리
- 식재료 목록이 비어있습 → 400 에랑, "생장고 사진을 인식해주세요" 안내
- 모델이 JSON 형식을 지키지 않았 → 파싱 실패 시 재시도 1회, 이후 실패 시 원문 통스트띌란 사용자에게 표시
- OpenRouter API 오랖 → PRD_01과 동일한 방식으로 처리

## 성공 기준
- 식재료 목록 제출 후 10쪽 이렄 레시피 1개 이상 반환
- 생성된 레시피의 재료가 입력 식재료 목록과 실제로 겁쎋는지(환각 방지) 확인

## 범위 (Out of Scope)
- 냉장고 이미지 인식 (→ PRD_01)
- 사용자 프로필 생성, 레시피 저장/즉겪찾기 (→ PRD_03)
- 영양 정보 산스
